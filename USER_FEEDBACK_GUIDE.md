## 🧪 사용자 테스트 시나리오

### **시나리오 1: 기본 검색 및 다운로드**

#### 1-1. 검색 테스트
```
Claude Desktop에서:
"reactor safety 검색해줘"
```

**체크포인트**:
- [ ] 검색 결과가 표시되나요?
- [ ] ML 번호가 정확하게 나오나요?
- [ ] 검색 시간이 너무 오래 걸리나요? (예상: 10-15초)

**피드백 항목**:
- 검색 속도: ⭐⭐⭐⭐⭐ (1-5점)
- 결과 관련성: ⭐⭐⭐⭐⭐ (1-5점)
- 표시 형식: 만족 / 보통 / 불만족

---

#### 1-2. 다운로드 테스트
```
"상위 3개 문서 다운로드해줘"
```

**체크포인트**:
- [ ] 다운로드가 성공했나요?
- [ ] 파일 위치를 찾을 수 있나요? (`downloaded_pdfs/`)
- [ ] PDF 파일이 정상적으로 열리나요?

**피드백 항목**:
- 다운로드 속도: ⭐⭐⭐⭐⭐
- 성공률: _개 / 3개
- 파일 크기: 적절 / 너무 큼 / 너무 작음

---

### **시나리오 2: Q&A 기능 테스트**

#### 2-1. 전체 문서 질문
```
"주요 안전 요구사항이 뭐야?"
"emergency procedures에 대해 알려줘"
```

**체크포인트**:
- [ ] 답변이 관련성 있나요?
- [ ] 인용 정보가 표시되나요? (예: [ML12345] Page 5)
- [ ] 답변 시간이 적절한가요? (예상: 첫 질문 5-10초, 이후 2-5초)

**피드백 항목**:
```
질문: "주요 안전 요구사항이 뭐야?"

답변 정확도: ⭐⭐⭐⭐⭐
답변 완성도: ⭐⭐⭐⭐⭐
인용 유용성: ⭐⭐⭐⭐⭐
응답 시간: _초 (만족 / 보통 / 느림)

개선 필요사항:
(자유롭게 작성)
```

---

#### 2-2. 특정 문서 질문
```
"ML081710326 문서에서 참석자 알려줘"
"In document ML030730366, what is the main topic?"
```

**체크포인트**:
- [ ] 특정 문서에서만 검색하나요?
- [ ] 다른 문서 내용이 섞이지 않나요?
- [ ] 에러 메시지가 명확한가요? (문서 없을 때)

**피드백 항목**:
- 문서 필터링 정확도: ⭐⭐⭐⭐⭐
- 에러 메시지 명확성: ⭐⭐⭐⭐⭐

---

### **시나리오 3: 페이지 인용 시스템**

#### 3-1. 인용 형식 확인
질문 후 답변에서 인용 정보를 확인하세요.

**예상 형식**:
```
[ML24275A095] Page 15 of 250 - Section 3.2 (Lines 450-475)
```

**체크포인트**:
- [ ] 페이지 번호가 표시되나요?
- [ ] 섹션 정보가 있나요?
- [ ] 인용 정보로 원문을 찾을 수 있나요?

**피드백 항목**:
```
인용 표시: 있음 / 없음 / 부분적
페이지 정확도: 정확 / 대략적 / 부정확
유용성: 매우 유용 / 보통 / 불필요

개선 아이디어:
(예: "섹션 제목을 더 명확하게", "라인 번호는 불필요" 등)
```
