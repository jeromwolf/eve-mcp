# EVE MCP 수동 테스트 가이드

## Claude Desktop에서 테스트하기

### 준비 사항
1. `npm install && npm run build` 실행
2. Claude Desktop 설정에 MCP 서버 추가
3. Claude Desktop 재시작

### 테스트 시나리오별 명령어

## 🔍 1단계: 논문 검색 테스트

### 테스트 1-1: 기본 검색
```
👤: "machine learning 논문 찾아줘"
✅ 예상: arXiv에서 10개 논문 목록
```

### 테스트 1-2: 검색 개수 지정
```
👤: "transformer 논문 3개만 검색해줘"
✅ 예상: 정확히 3개 결과
```

### 테스트 1-3: 특정 사이트 검색
```
👤: "PubMed에서 COVID-19 백신 논문 5개 찾아줘"
✅ 예상: PubMed 검색 결과 5개
```

### 테스트 1-4: 복잡한 검색어
```
👤: "attention is all you need 논문 검색"
✅ 예상: Transformer 원논문 포함된 결과
```

## 📥 2단계: PDF 다운로드 테스트

### 테스트 2-1: URL로 다운로드
```
👤: "https://arxiv.org/pdf/1706.03762.pdf 다운로드해줘"
✅ 예상: 
- 파일명: 1706.03762.pdf
- 제목, 저자, 페이지 수 표시
- "이제 질문할 수 있습니다" 메시지
```

### 테스트 2-2: 검색 후 다운로드
```
👤: "BERT 논문 검색해줘"
👤: "첫 번째 논문 다운로드해줘"
✅ 예상: 검색 결과의 첫 번째 PDF 다운로드
```

### 테스트 2-3: 중복 다운로드
```
👤: "방금 그 논문 다시 다운로드해줘"
✅ 예상: "이미 다운로드됨" 메시지
```

## 💬 3단계: PDF 질의응답 테스트

### 테스트 3-1: 기본 질문
```
👤: "이 논문의 핵심 아이디어가 뭐야?"
✅ 예상: 최근 다운로드한 PDF에서 답변
```

### 테스트 3-2: 구체적 질문
```
👤: "attention mechanism이 어떻게 작동해?"
✅ 예상: 관련 단락 찾아서 설명
```

### 테스트 3-3: 파일명 지정
```
👤: "1706.03762.pdf에서 self-attention 설명 찾아줘"
✅ 예상: 특정 파일에서 검색
```

### 테스트 3-4: 한국어 질문
```

✅ 예상: limitation/drawback 관련 내용
```

## 🐛 문제 발생 시

1. **MCP 연결 안됨**: 
   - Claude Desktop 완전 재시작
   - 설정 파일 경로 확인

2. **검색 결과 없음**:
   - 네트워크 연결 확인
   - API 응답 확인

3. **PDF 다운로드 실패**:
   - URL 형식 확인
   - 파일 크기 확인 (50MB 제한)

4. **질의응답 안됨**:
   - PDF 다운로드 여부 확인
   - 텍스트 추출 성공 확인
